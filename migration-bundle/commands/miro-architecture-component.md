# Component Diagram Templates

Create detailed component relationships. Choose a template:

## 1. Class Diagram Style
**Command:** `/miro-architecture-component-class`

UML-style class relationships. Best for object-oriented design.

**Example:** Domain model, class structure, entity relationships

---

## 2. Module Dependencies
**Command:** `/miro-architecture-component-module`

Module imports and dependencies. Best for code organization.

**Example:** Package structure, import graph, dependency tree

---

## 3. Container Diagram
**Command:** `/miro-architecture-component-container`

Containers with technology stack. Best for deployment architecture.

**Example:** Web app + API + DB, Docker containers, deployment units

---

Tell the user to choose a template or describe their component structure.
